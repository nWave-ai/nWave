# nw-data-engineer

Use for database technology selection, data architecture design, query optimization, schema design, security implementation, and governance guidance. Provides evidence-based recommendations across RDBMS and NoSQL systems.

**Wave:** Other
**Model:** inherit
**Max turns:** 30
**Tools:** Read, Write, Edit, Glob, Grep, Bash

## Skills

- [data-architecture-patterns](../../../nWave/skills/data-engineer/data-architecture-patterns.md) — Data architecture patterns (warehouse, lake, lakehouse, mesh), ETL/ELT pipelines, streaming architectures, scaling strategies, and schema design patterns
- [database-technology-selection](../../../nWave/skills/data-engineer/database-technology-selection.md) — Database comparison catalogs, RDBMS vs NoSQL selection criteria, CAP/ACID/BASE theory, OLTP vs OLAP, and technology-specific characteristics
- [query-optimization](../../../nWave/skills/data-engineer/query-optimization.md) — SQL and NoSQL query optimization techniques, indexing strategies, execution plan analysis, JOIN algorithms, cardinality estimation, and database-specific query patterns
- [security-and-governance](../../../nWave/skills/data-engineer/security-and-governance.md) — Database security (encryption, access control, injection prevention), data governance (lineage, quality, MDM), and compliance frameworks (GDPR, CCPA, HIPAA)
