# Commands

| Command | Description | Agents | Arguments |
| --- | --- | --- | --- |
| `/nw:deliver` | Orchestrates the full DELIVER wave end-to-end (roadmap > execute-all > finalize). Use when all prior waves are complete and the feature is ready for implementation. | [nw-platform-architect](../agents/nw-platform-architect.md), [nw-software-crafter](../agents/nw-software-crafter.md), [nw-software-crafter-reviewer](../agents/nw-software-crafter-reviewer.md), [nw-solution-architect](../agents/nw-solution-architect.md), [nw-troubleshooter](../agents/nw-troubleshooter.md) | [feature-description] - Example: "Implement user authentication with JWT" |
| `/nw:design` | Designs system architecture with C4 diagrams and technology selection. Use when defining component boundaries, choosing tech stacks, or creating architecture documents. | [nw-platform-architect](../agents/nw-platform-architect.md), [nw-solution-architect](../agents/nw-solution-architect.md) | [component-name] - Optional: --residuality (enable residuality analysis for complex/critical systems) |
| `/nw:devops` | Designs CI/CD pipelines, infrastructure, observability, and deployment strategy. Use when preparing platform readiness for a feature. | [nw-acceptance-designer](../agents/nw-acceptance-designer.md), [nw-platform-architect](../agents/nw-platform-architect.md) | [deployment-target] - Optional: --environment=[staging|production] --validation=[full|smoke] |
| `/nw:diagram` | Generates C4 architecture diagrams (context, container, component) in Mermaid or PlantUML. Use when creating or updating architecture visualizations. | [nw-solution-architect](../agents/nw-solution-architect.md) | [diagram-type] - Optional: --format=[mermaid|plantuml|c4] --level=[context|container|component] |
| `/nw:discover` | Conducts evidence-based product discovery through customer interviews and assumption testing. Use at project start to validate problem-solution fit. | [nw-product-discoverer](../agents/nw-product-discoverer.md), [nw-product-owner](../agents/nw-product-owner.md) | [product-concept] - Optional: --interview-depth=[overview|comprehensive] --output-format=[md|yaml] |
| `/nw:discuss` | Conducts UX journey design and requirements gathering through interactive discovery. Use when starting feature analysis, defining user stories, or creating acceptance criteria. | [nw-product-owner](../agents/nw-product-owner.md), [nw-solution-architect](../agents/nw-solution-architect.md) | [feature-name] - Optional: --phase=[journey|requirements] --interactive=[high|moderate] --output-format=[md|yaml] |
| `/nw:distill` | Creates E2E acceptance tests in Given-When-Then format from requirements and architecture. Use when preparing executable specifications before implementation. | [nw-acceptance-designer](../agents/nw-acceptance-designer.md), [nw-software-crafter](../agents/nw-software-crafter.md) | [story-id] - Optional: --test-framework=[cucumber|specflow|pytest-bdd] --integration=[real-services|mocks] |
| `/nw:document` | Creates evidence-based documentation following DIVIO/Diataxis principles. Use when writing tutorials, how-to guides, reference docs, or explanations. | [nw-documentarist](../agents/nw-documentarist.md), [nw-documentarist-reviewer](../agents/nw-documentarist-reviewer.md), [nw-researcher](../agents/nw-researcher.md), [nw-researcher-reviewer](../agents/nw-researcher-reviewer.md) | [topic/component] - Optional: --type=[tutorial|howto|reference|explanation] --research-depth=[overview|detailed|comprehensive|deep-dive] |
| `/nw:execute` | Dispatches a single roadmap step to a specialized agent for TDD execution. Use when implementing a specific step from a roadmap.yaml plan. | [nw-researcher](../agents/nw-researcher.md), [nw-software-crafter](../agents/nw-software-crafter.md) | [agent] [project-id] [step-id] - Example: @nw-software-crafter "auth-upgrade" "01-01" |
| `/nw:finalize` | Archives a completed feature to docs/evolution/ and cleans up workflow files. Use after all implementation steps pass and mutation testing completes. | [nw-acceptance-designer](../agents/nw-acceptance-designer.md), [nw-platform-architect](../agents/nw-platform-architect.md), [nw-product-owner](../agents/nw-product-owner.md), [nw-researcher](../agents/nw-researcher.md), [nw-software-crafter](../agents/nw-software-crafter.md), [nw-solution-architect](../agents/nw-solution-architect.md) | [agent] [project-id] - Example: @platform-architect "auth-upgrade" |
| `/nw:forge` | Creates new specialized agents using the 5-phase workflow (ANALYZE > DESIGN > CREATE > VALIDATE > REFINE). Use when building a new AI agent or validating an existing agent specification. | [nw-agent-builder](../agents/nw-agent-builder.md) | [agent-name] - Optional: --type=[specialist|reviewer|orchestrator] --pattern=[react|reflection|router] |
| `/nw:mikado` | [EXPERIMENTAL] Complex refactoring roadmaps with visual tracking | [nw-software-crafter](../agents/nw-software-crafter.md) | [refactoring-goal] - Optional: --complexity=[simple|moderate|complex] --visualization=[tree|graph] |
| `/nw:mutation-test` | Runs feature-scoped mutation testing to validate test suite quality. Use after implementation to verify tests catch real bugs (kill rate >= 80%). | [nw-software-crafter](../agents/nw-software-crafter.md) | [project-id] - Optional: --threshold=[75|80|85] --language=[auto|python|java|javascript] |
| `/nw:refactor` | Applies the Refactoring Priority Premise (RPP) levels L1-L6 for systematic code refactoring. Use when improving code quality through structured refactoring passes. | [nw-software-crafter](../agents/nw-software-crafter.md) | [target-class-or-module] - Optional: --level=[1-6] --method=[extract|inline|rename|move] --scope=[method|class|module] |
| `/nw:research` | Gathers knowledge from web and files, cross-references across multiple sources, and produces cited research documents. Use when investigating technologies, patterns, or decisions that need evidence backing. | [nw-researcher](../agents/nw-researcher.md) | [topic] - Optional: --research_depth=[overview|detailed|comprehensive|deep-dive] --skill-for=[agent-name] |
| `/nw:review` | Dispatches an expert reviewer agent to critique workflow artifacts. Use when a roadmap, implementation, or step needs quality review before proceeding. | [nw-platform-architect](../agents/nw-platform-architect.md), [nw-platform-architect-reviewer](../agents/nw-platform-architect-reviewer.md), [nw-software-crafter](../agents/nw-software-crafter.md), [nw-software-crafter-reviewer](../agents/nw-software-crafter-reviewer.md), [nw-solution-architect](../agents/nw-solution-architect.md), [nw-solution-architect-reviewer](../agents/nw-solution-architect-reviewer.md) | [agent] [artifact-type] [artifact-path] - Example: @software-crafter task "roadmap.yaml" |
| `/nw:roadmap` | Creates a phased roadmap.yaml for a feature goal with acceptance criteria and TDD steps. Use when planning implementation steps before execution. | [nw-product-owner](../agents/nw-product-owner.md), [nw-software-crafter](../agents/nw-software-crafter.md), [nw-solution-architect](../agents/nw-solution-architect.md) | [agent] [goal-description] - Example: @solution-architect "Migrate to microservices" |
| `/nw:root-why` | Root cause analysis and debugging | [nw-troubleshooter](../agents/nw-troubleshooter.md) | [problem-description] - Optional: --depth=[3|5|7-whys] --output=[text|diagram|report] |
